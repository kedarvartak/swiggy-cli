# Workflow Flow

## Product Direction

Swiggy workflows should not behave like an n8n-style builder for regular users.
Users should not need to understand MCP tools or manually assemble low-level steps.

The right model is a workflow marketplace built around repeatable outcomes.
Users browse reusable workflow templates, configure simple preferences, save them as their own, run them anytime, watch transparent execution, and approve the final action.

## Core Flow

1. User opens Workflows
2. Sees marketplace cards
3. Opens one template
4. Configures simple preferences
5. Saves as My workflow
6. Runs it anytime
7. Watches transparent execution
8. Approves final action

## Principle

MCP tools stay underneath the surface.
The UI should expose goals, preferences, guardrails, and execution visibility instead of raw tool selection.
